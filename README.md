# ⚖️ Онлайн Суд - Многопользовательская игра

Полноценная браузерная многопользовательская судебная игра с real-time взаимодействием через WebSocket.

## 🚀 Быстрый старт

### Локальный запуск

```bash
# 1. Установка зависимостей
npm install

# 2. Сборка фронтенда
npm run build

# 3. Запуск сервера
npm start
```

Откройте `http://localhost:3000` и начинайте играть!

### Development режим

**Терминал 1 - Frontend:**
```bash
npm run dev
```

**Терминал 2 - Backend:**
```bash
npm run server
```

Frontend: `http://localhost:5173`  
Backend: `http://localhost:3000`

## 🌐 Deploy на Railway

### Автоматический deploy

1. Пуш в GitHub:
```bash
git add .
git commit -m "Ready for deploy"
git push
```

2. На [railway.app](https://railway.app):
   - Нажмите "New Project"
   - Выберите "Deploy from GitHub repo"
   - Выберите ваш репозиторий
   - Railway автоматически соберёт и задеплоит проект

### Переменные окружения (опционально)

Для Discord OAuth добавьте в Railway:
```
VITE_DISCORD_CLIENT_ID=ваш_discord_client_id
```

Подробнее в [DISCORD_SETUP.md](DISCORD_SETUP.md)

## 🎮 Как играть

### 1️⃣ Вход в игру
- Нажмите **"Войти как гость"**
- Вам автоматически присвоится имя `Гость#12345678`
- Или используйте **"Войти через Discord"** (если настроен)

### 2️⃣ Добавить друзей
1. Нажмите **"Друзья"** в шапке
2. Попросите друга сказать его имя
3. Введите имя друга (например: `Гость#87654321`)
4. Нажмите **"+"**

### 3️⃣ Создать лобби
1. Нажмите **"Создать лобби"**
2. Получите уникальный ID лобби
3. Нажмите **"Пригласить друзей"**
4. Выберите друга из списка
5. Друг получит уведомление

### 4️⃣ Начать игру
1. Дождитесь **минимум 3 игроков** в лобби
2. Хост нажимает **"Начать игру"**
3. Система генерирует случайное дело

### 5️⃣ Выбрать роль
Каждый игрок выбирает:
- **Судья** (1) - выносит вердикт
- **Прокурор** (1) - обвиняет
- **Адвокат** (1) - защищает
- **Истец** (1+) - пострадавший

### 6️⃣ Разобрать дело
1. Изучите материалы дела
2. Обсудите в Discord (голосовой чат)
3. Используйте **"Заметки"** для записей
4. Прокурор обвиняет, адвокат защищает

### 7️⃣ Вынести вердикт
1. **Судья** выбирает: Виновен / Не виновен
2. При обвинении назначает наказание
3. Нажимает **"Огласить вердикт"**
4. Дело завершается

## 🔧 Настройка Discord OAuth

Подробная инструкция в [DISCORD_SETUP.md](DISCORD_SETUP.md)

**Кратко:**
1. Создайте приложение на [Discord Developers](https://discord.com/developers/applications)
2. Скопируйте Client ID
3. Создайте `.env` файл:
```env
VITE_DISCORD_CLIENT_ID=ваш_client_id
```
4. Перезапустите приложение

## 📊 WebSocket API

### События клиент → сервер
- `login:guest` - вход как гость
- `login:discord` - вход через Discord
- `friend:add` - добавить друга
- `friend:list` - получить друзей
- `friend:remove` - удалить друга
- `lobby:create` - создать лобби
- `lobby:invite` - пригласить в лобби
- `lobby:join` - присоединиться
- `lobby:leave` - покинуть лобби
- `game:start` - начать игру
- `game:selectRole` - выбрать роль
- `game:verdict` - вынести вердикт

### События сервер → клиент
- `friend:added` - друг добавлен
- `friend:removed` - друг удалён
- `lobby:updated` - лобби обновлено
- `lobby:invited` - приглашение получено
- `game:started` - игра началась
- `game:finished` - игра завершена

## 🐛 Решение проблем

### Кнопка "Войти как гость" не работает

**Проверьте:**
1. Запущен ли backend сервер (`npm start` или `npm run server`)
2. Нет ли ошибок в консоли браузера (F12)
3. Доступен ли сервер на `localhost:3000` или `localhost:5173`

**Решение:**
```bash
# Остановите все процессы
# Запустите заново:
npm run build
npm start
```

### Discord OAuth не работает

**Проверьте:**
1. Добавлен ли `VITE_DISCORD_CLIENT_ID` в `.env`
2. Правильный ли Client ID
3. Добавлен ли Redirect URI в Discord приложении

**Подробнее:** [DISCORD_SETUP.md](DISCORD_SETUP.md)

### Друг не получает приглашение

**Проверьте:**
1. Друг онлайн и подключен к серверу
2. Правильно ли добавлен друг (проверьте имя)
3. Друг не находится уже в другом лобби

### Railway build failed

**Ошибка с Node.js версией:**
- Проверьте `.node-version` - должно быть `22`
- Проверьте `nixpacks.toml` - должно быть `nodejs_22`
